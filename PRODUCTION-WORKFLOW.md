# PadelStats — Workflow vers publication

## Ordre retenu

### 1. Stabiliser totalement le produit
- connexion / déconnexion
- ajout de match
- reconnexion
- persistance propre
- aucun bouton mort

### 2. Finaliser la logique coûts
- coûts terrains
- accessoires séparés
- filtres propres
- historique fiable

### 3. Rendre l’UI vraiment mobile/app-like
- navigation plus propre
- écrans mieux hiérarchisés
- sensation d’app plus nette
- moins “site web dans un téléphone”

### 4. Fiabiliser la base publiable
- build qui s’ouvre sans bug
- assets propres
- manifest / icône / splash
- base Android/iPhone exploitable

### 5. Préparer la publication store
- nom final
- icône finale
- captures d’écran
- description store
- politique de confidentialité si nécessaire
- bundle/package propre
- comptes développeur

### 6. Tester avant soumission
- Android
- iPhone
- cas réels
- refresh
- offline léger si utile
- régression générale

## Priorité réelle
Avant publication, le plus important est :
- stabilité
- fluidité mobile
- cohérence du flux complet

## Cible immédiate
1. consolider le produit
2. rendre l’interface plus app-like
3. refaire un build propre
4. ensuite seulement publication
